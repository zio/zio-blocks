# Issue #802: Enhance toString of core data types

**Captured**: 2026-01-24
**URL**: https://github.com/zio/zio-blocks/issues/802
**Author**: jdegoes
**Labels**: $500, Bounty

## Description

Add custom `toString` implementations across the zio-blocks schema library to provide human-readable, debugger-friendly representations. Each type's format should draw from established prior art and match existing project conventions where applicable.

### Motivation

Currently, many types use default case class `toString`, which produces verbose, hard-to-read output that makes debugging difficult. For example:

```
TypeName(Namespace(List(scala, collection, immutable)),List,List(TypeName(Namespace(List(scala)),Int,List())))
```

Should instead be:

```
scala.collection.immutable.List[scala.Int]
```

## Specifications Summary

1. **TypeName[A]** - Valid Scala type syntax (e.g., `scala.Int`, `scala.Option[scala.String]`)
2. **DynamicOptic** - Match path interpolator syntax (e.g., `.name`, `<Some>`, `[0]`, `{*}`)
3. **Optic Types** - Match `$` macro syntax (e.g., `Lens(_.name)`, `Traversal(_.users.each.email)`)
4. **Reflect Types** - SDL-style fully recursive expansion
5. **Schema[A]** - Delegate to Reflect with wrapper
6. **Term** - Field name with fully-expanded type
7. **DynamicValue** - EJSON format with records, variants, typed primitives
8. **DynamicPatch/Patch** - Line-oriented diff format
9. **Json** - Delegate to existing `print` method

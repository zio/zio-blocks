# Project-Specific Instructions for ZIO Blocks Modern

## Skill Introspection Phase

When executing a skill and encountering failures, deviations from the skill's instructions, or missing guidance:

1. **Identify the issue**: Note what went wrong, what the skill should have covered, or how the behavior deviated from documented instructions
2. **Complete the task first**: Finish the current work before refactoring
3. **Suggest skill repair**: After the skill execution completes, recommend updates to the skill to address the identified issues
4. **Update the skill**: Apply the improvements following TDD principles (baseline test → minimal fix → verify)

This keeps skills accurate and prevents the same gaps from affecting future work.
